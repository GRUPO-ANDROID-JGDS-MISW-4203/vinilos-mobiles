# Reglas de Equipo y GitFlow

## Flujo base

- `main`: estable o release.
- `develop`: integracion.
- `feature/*`: implementacion por historia o tarea.

## Reglas

- No hacer cambios grandes sin necesidad funcional.
- Mantener trazabilidad a historia o issue cuando aplique.
- Los cambios deben ser revisables y reversibles.
- La documentacion no debe separarse del estado real del sprint.

## Evidencia

En este proyecto la evidencia de reunion, pruebas, README y wiki tambien hacen parte del trabajo.
