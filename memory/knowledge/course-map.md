# Mapa del Curso

## Semana 1

- Ecosistema movil.
- Tipos de apps.
- Inception del proyecto.
- Definicion del MVP.
- Planeacion, backlog, UX/UI y reglas del equipo.

## Semana 2

- Kotlin como lenguaje principal.
- Componentes Android.
- Activities, fragments, temas y estilos.
- Primer enfoque de calidad y pruebas.

## Semana 3 en adelante

- Consumo de APIs REST.
- MVVM.
- Repository y Service Adapter.
- Espresso y pruebas instrumentadas.
- Integracion por historias.

## Semanas avanzadas

- Corrutinas y asincronia.
- Cache y estructuras de datos.
- Carga eficiente de imagenes.
- Fragmentacion y pruebas cloud.
- Firebase Test Lab, ADB y matrices de dispositivos.
- Piramide de pruebas y estrategia TNT.
- Micro-optimizaciones, memoria y prevencion de ANRs.

## Implicacion para el repo

El proyecto no debe verse solo como una app que funciona en un telefono del equipo.
Debe demostrar proceso de ingenieria movil: arquitectura, pruebas, documentacion, navegacion, rendimiento razonable y evidencia por sprint.
