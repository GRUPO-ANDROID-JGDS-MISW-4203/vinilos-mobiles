# Definition of Done

Una tarea se considera terminada solo si:

- el codigo implementa el comportamiento esperado,
- compila,
- respeta arquitectura,
- tiene validacion razonable,
- mantiene navegacion y estados coherentes,
- y la documentacion necesaria no queda desalineada.

## Para historias de usuario

Adicionalmente:

- la historia es trazable a su capa tecnica,
- existe evidencia de prueba,
- y el cambio puede integrarse al sprint sin romper el resto del MVP.
