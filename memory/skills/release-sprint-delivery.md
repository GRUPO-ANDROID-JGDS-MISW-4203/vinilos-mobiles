# Skill: Release y Entrega por Sprint

## Objetivo

Guiar el cierre de iteraciones con foco academico y tecnico.

## Entregables esperados

- codigo funcional,
- estrategia de pruebas,
- artefactos de prueba,
- README util,
- APK o evidencia de build,
- documentos de arquitectura/diseno,
- trazabilidad por historia.

## Checklist de cierre tecnico

- Compila el proyecto.
- Pasan las pruebas relevantes.
- La navegacion base no se rompio.
- El README sigue permitiendo levantar el proyecto.
- La documentacion refleja estado real.

## Checklist de cierre academico

- La historia esta documentada por sprint.
- Existe evidencia de pruebas.
- Existe evidencia de reunion o seguimiento si el sprint lo requiere.
- El entregable es trazable a HU y rubrica.

## Riesgos frecuentes

- Codigo listo pero sin evidencia de prueba.
- Wiki adelantada al codigo real.
- README desactualizado.
- APK faltante o build no validado.

## Regla final

En este curso "terminado" no significa solo compilar: significa codigo, pruebas, evidencia y documentacion consistentes.
